# Loka website

Static one-page site for Loka, generated by a small zero-dependency Node
script. The generator is a deliberate placeholder: the long-term plan is for
Loka itself to produce this site, so all content lives as data and only the
rendering layer is disposable.

## Build

```sh
node site/build.mjs
```

Output goes to `site/dist/` (gitignored). Open `site/dist/index.html` in a
browser to preview.

## Updating content (the incremental loop)

Everything on the page comes from [`data/site.json`](data/site.json) — hero
copy, the code sample, examples, features, the platform table, and quickstart
steps. To update the site, edit that file and rebuild. No HTML editing needed
for routine updates.

### Adding an example / screenshot

1. Add an entry to `examples.items` in `data/site.json` (slug, name,
   description, source path).
2. Drop a real screenshot at `static/images/examples/<slug>.png`
   (also `.jpg`/`.jpeg`/`.webp`; 4:3 looks best, cards crop with
   `object-fit: cover`).
3. Rebuild. Until the screenshot exists, the build generates an SVG
   placeholder automatically, so it is fine to ship entries before you have
   pictures.

### Adding a hardware / workflow photo

Same mechanism as examples: add an entry to `hardware.gallery` in
`data/site.json` (slug, name, caption) and drop the photo at
`static/images/hardware/<slug>.jpg` (or png/jpeg/webp). Placeholders are
generated until the photo exists.

### Everything else

- Styles: `src/styles.css` — light/dark themes via CSS custom properties.
- Hero artwork and the hardware photo are pulled from the repo-level
  `assets/` directory at build time.
- Deployment: `.github/workflows/site.yml` builds and publishes to GitHub
  Pages on pushes to `main` that touch `site/` or `assets/`.
  (One-time setup: repository Settings → Pages → Source: GitHub Actions.)
