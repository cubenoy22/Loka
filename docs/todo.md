# 現状設計の弱み・懸念事項リスト（2025-05-03 時点）

1. **UI の動的再構築・差分更新が未対応**

   - App::rerender は setPage 時のみ呼ばれ、状態変化による UI 再構築・再レンダリングは行わない
   - 動的な UI 切り替えや部分的な再描画が必要な場合は、Page の中で Page を切り替える等の工夫が必要

2. **複雑な UI ツリー・ネスト構造の拡張性**

   - Page の中で Page を持つことで if 分岐や動的 UI は実現できるが、複雑なネストや再利用性の高い UI パターンは今後の設計課題

3. **イベント伝播・コールバックの仕組み**

   - 現状は onClick 等の関数ポインタのみ。より複雑なイベント伝播や親子間通信は今後の拡張テーマ

4. **レイアウト・スタイルの柔軟性**

   - レイアウトやスタイル指定は未実装。今後 Grid/Row/Column 的な部品やスタイル指定 API の追加が必要

5. **テスト性・デバッグ性**

   - UI 状態のダンプやスナップショットテスト、UI ツリーの可視化などは未対応

6. **例外安全性・エラー処理**

   - new 失敗時や不正な Prop 参照など、エラー処理・例外安全性は今後強化の余地あり

7. **C++98 の制約**

   - スマートポインタやラムダ式が使えないため、手動 delete や関数ポインタ設計に注意が必要
   - テンプレートや RTTI の限界も意識

8. **Renderer の仮実装**
   - 実際の描画処理はまだ仮実装。プラットフォームごとの実装や高機能化は今後の課題
